Auth

Up 
post /login
An existing user logs in to his account. (loginPost)
A JSON object contains all the details about the logged in user.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body Login-user (required)
Body Parameter — Item to Get

Responses

200

the user logged in.

400

wrong input.

404

the user does not exist.

500

the server has encountered a situation it doesnt know how to handle.
